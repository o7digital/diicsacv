# DIICSA Astro Migration

This migration must preserve the current WordPress mirror design first, then replace it page by page.

## Current Baseline

- Branch: `dev`
- Runtime: Astro static output
- Source mirror: `diicsacv.com/`
- Astro public copy: generated by `npm run sync:mirror`
- Do not edit `public/`; it is generated.
- Production preview: Vercel CLI after each committed step.

## Parity Rule

Before replacing any mirrored HTML with native Astro:

1. Keep the original mirrored route available until the replacement is visually checked.
2. Build successfully with `npm run build`.
3. Verify at least:
   - `/`
   - `/diicsacv.com/`
   - the page being migrated
   - one representative asset from that page
4. Deploy with Vercel CLI.
5. Do not delete Elementor, Smart Slider, Hummingbird, font, or upload assets until every page using them has an Astro-native replacement.

## Shared WordPress/Elementor Pieces

These Elementor CSS IDs are common across most pages:

- `992`: global kit/base styling
- `997`: header
- `1006`: footer
- `1013`: single page wrapper/breadcrumb layout

The home page additionally uses:

- `1015`: home page Elementor layout
- Smart Slider 3 assets under `diicsacv.com/wp-content/plugins/smart-slider-3/`

## Recommended Migration Order

1. `vision`: simple content page with one main image.
2. `mision`, `politica`, `capacitacion`, `historia`: same page family.
3. Service pages: `edificacion-y-rehabilitacion`, `obra-electrica-alta-y-media`, `obra-de-procesos-mecanica`, `obra-civil-en-general`.
4. `clientes`, `certificados`, `presencia`.
5. `contacta-con-nosotras`.
6. Home page last because it contains Smart Slider markup and the largest design surface.

## Next Safe Step

Create an Astro-native layout shell for one hidden test route, for example `/astro/vision/`, using the same assets and visual measurements. Keep `/diicsacv.com/vision/` as the reference until the replacement matches.
